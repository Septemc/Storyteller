# Storyteller 项目开发任务清单（整合版）

**更新日期**: 2026-03-10  
**版本**: v0.3.1

---

## 一、项目目标

- 构建一个 **FastAPI + SQLite + 原生前端** 的互动叙事/跑团系统。
- 打通“角色库 + 世界书 + 副本脚本 + 会话状态 + RAG 检索 + 大模型生成”的完整闭环。
- 在保证稳定性的前提下，逐步将向量检索从 TF-IDF 升级到 Transformer Embedding 主链路。

## 六、验收标准（量化）

- [ ] 检索质量：Top-3 命中率较 TF-IDF 提升 ≥ 20%
- [ ] 性能：P95 检索耗时 ≤ 500ms（当前数据规模）
- [ ] 稳定性：Embedding 异常时自动降级，可用率 ≥ 99.9%
- [ ] 工程性：可配置、可回滚、可重建、可监控

---

## 七、风险与应对

- [ ] 依赖体积增加（`torch/sentence-transformers`）→ 采用可选安装与镜像缓存
- [ ] 模型维度变化导致缓存不兼容 → 模型切换触发强制重建
- [ ] 外部 API 限流/超时 → 超时降级 + 重试 + 熔断
- [ ] 检索规模增长导致余弦计算变慢 → 提前规划 ANN 索引
