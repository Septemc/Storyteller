## 🚀 主要改动回顾（代码侧）

### 1) 脚本持久化系统

- 数据层：新增 `scripts` 表，`session_state` 增加 `current_script_id`
- API 层：新增脚本 CRUD（`/api/scripts`）
- 前端层：副本页自动同步脚本，并联动会话上下文
- 迁移层：提供 `backend/scripts/migrate_add_script_table.py`

**价值**：实现“副本脚本 → 会话绑定 → 持久化保存”的闭环能力。

---

### 2) 稳定性修复与角色注入增强

- 修复 `routes_story.py` 语法问题，解决页面转圈与后端不可用
- 角色字段解析支持多键回退（`name/f_name/f_nickname/...`）
- 主角识别类型扩展（`pc/player/protagonist/main/hero`）
- 增加 query-aware 角色筛选/评分与 Prompt 注入“角色库（节选）”

**价值**：剧情生成可稳定注入主角 + NPC 角色语义，回答相关问题的可解释性明显增强。

---

### 3) 世界书与向量缓存同步优化

- 世界书导入/更新/删除与 `worldbook_embeddings` 实现同步联动
- 前端改为增量同步（仅提交变化条目）
- 后端支持批量向量计算，降低长文本导入卡顿
- 修复常见问题：内容刷新丢失、删除后回弹、page_size 超限、同步失败无提示

**价值**：显著提升同步稳定性与性能，减少无效向量重复计算。

---

### 4) 向量化架构能力升级（基础设施）

- `EmbeddingEngine` 已支持多 Provider：`tfidf / sentence_transformers / openai`
- 已具备失败降级链路（异常回退）
- 项目依赖已包含 `sentence-transformers`

**价值**：为“从 TF-IDF 向 Transformer 主链路切换”提供可落地基础。

---

## ✅ 2026-03-10 更新摘要

### 世界书与导入体验
- 世界书删除接口补齐并修复路由匹配顺序（`/worldbook/all`、`/worldbook/category`、`/worldbook/{entry_id}`）
- 世界书导入改为后台计算向量（失败自动回退 TF-IDF），前端新增“正在导入”提示
- 导入失败提示修正（区分 JSON 解析失败 / localStorage 超限）
- 新增条目开关 UI（enabled/disable）并单条同步到后端
- 导入条目 ID 随机化，避免 `1/2/3` 简单 ID
- 删除世界书/模块增加“删除成功”提示

### 认证与稳定性
- 登录/注册哈希改为 `pbkdf2_sha256`，兼容旧 bcrypt 哈希

### 向量化主链路
- 默认检索切换到 sentence-transformers（失败自动回滚 TF-IDF）
