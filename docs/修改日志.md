## 🔧 核心修改说明

### 1. backend/api/routes_worldbook.py

#### 修改 1: 批量向量计算优化

**问题**:
- 向量计算是串行的，每个条目计算一次
- 每次计算都要重新拟合 TF-IDF 词汇表
- 长文本（7000+ 字符）计算慢

**解决方案**:
- 批量计算所有条目的向量
- 一次拟合 TF-IDF 词汇表
- 直接保存向量到数据库

**代码变化**:
```python
# 修改前：串行计算
for entry in entries_to_embed:
    content = f"{entry.title} {entry.content}"
    embedding = retriever.compute_entry_embedding(entry, use_cache=False)
    # 每个条目都重新拟合词汇表

# 修改后：批量计算
texts_to_embed = []
for entry in entries_to_embed:
    content = f"{entry.title} {entry.content}"
    texts_to_embed.append((entry.entry_id, content))

# 一次计算所有向量
all_vectors = retriever.compute_batch_embeddings(
    [text for _, text in texts_to_embed],
    max_dim=500
)

# 批量保存到数据库
for i, (entry_id, _) in enumerate(texts_to_embed):
    vector = all_vectors[i]
    # 直接保存，跳过重复计算
    existing_emb = db.query(models.WorldbookEmbedding).filter(...).first()
    if existing_emb:
        existing_emb.vector = vector.tobytes()
    else:
        db.add(models.WorldbookEmbedding(...))
```

#### 修改 3: 增量同步优化 ⭐ 新增

**问题**: 
- 每次保存都同步所有条目（11 条）
- 大量数据传输和向量计算导致卡顿
- 长文本（7000+ 字符）向量计算慢

**解决方案**: 
1. 先获取数据库条目列表
2. 比较差异，只同步变化的条目
3. 添加同步锁，防止重复请求

**代码变化**:
```javascript
// 修改前：同步所有条目
async function syncToDatabaseSilent() {
  const entries = [];
  Object.entries(world.categories).forEach(([catName, catEntries]) => {
    catEntries.forEach(entry => {
      entries.push({...});  // 所有条目
    });
  });
  fetch('/api/worldbook/import', {body: JSON.stringify(entries)});
}

// 修改后：增量同步
async function syncToDatabaseSilent() {
  // 1. 获取数据库列表
  const listResponse = await fetch('/api/worldbook/list?page_size=1000');
  const dbEntries = listResponse.items || [];
  const dbMap = new Map();
  dbEntries.forEach(e => dbMap.set(e.entry_id.replace('WB_', ''), e));
  
  // 2. 只同步变化的条目
  const entriesToSync = [];
  Object.entries(world.categories).forEach(([catName, catEntries]) => {
    catEntries.forEach(entry => {
      const dbEntry = dbMap.get(entry.id);
      const needsSync = !dbEntry || 
                       dbEntry.title !== entry.title ||
                       dbEntry.content !== (entry.content || '') ||
                       dbEntry.importance !== (entry.importance || 0.5);
      if (needsSync) {
        entriesToSync.push({...});
      }
    });
  });
  
  // 3. 添加同步锁
  if (syncPromise) return;  // 防止重复同步
  syncPromise = fetch('/api/worldbook/import', {...});
  await syncPromise;
  syncPromise = null;
}
```

**效果**:
- 新增条目：只同步 1 条（而不是全部 11 条）
- 更新条目：只同步变化的 1 条
- 性能提升：90%+ 减少数据传输和向量计算

### 3. backend/core/rag.py

#### 新增：compute_batch_embeddings 方法

**作用**: 批量计算文本向量，用于导入接口快速计算

**代码**:
```python
def compute_batch_embeddings(
    self,
    texts: List[str],
    max_dim: int = 500,
) -> List:
    """批量计算文本的向量（用于导入时快速计算）"""
    if not texts:
        return []
    
    # 对于 TF-IDF，使用批量计算
    if self.embedding_config.provider == "tfidf":
        # 重新拟合 TF-IDF 模型以包含新文本
        from .embeddings import TFIDFVectorizer
        tfidf = TFIDFVectorizer(max_features=max_dim)
        normalized_texts = [t.lower().strip() for t in texts]
        tfidf.fit(normalized_texts)
        return tfidf.transform(normalized_texts)
    
    # 对于其他引擎，使用通用方法
    return [self.engine.compute_single(text) for text in texts]
```

4. **用户误以为成功**：模态框关闭了，但实际同步失败

1. 修改 `page_size=100`（符合后端限制）
2. 同步失败时显示 alert 弹窗
3. 抛出异常，让调用者知道同步失败
4. 添加详细错误信息（HTTP 状态码）

**代码变化**:
```javascript

// catch 中静默失败

// 修改后：page_size=100，显示详细错误
const listResponse = await fetch('/api/worldbook/list?page_size=100');
if (!listResponse.ok) {
  const errorText = await listResponse.text();
  throw new Error(`获取列表失败：HTTP ${listResponse.status} - ${errorText}`);
}
```

## 🔮 后续改进方向

### 向量化方案升级

**当前方案**: TF-IDF 基础向量化（768 维）
- ✅ 优点：无需外部依赖，计算快速
- ❌ 缺点：无法理解语义，仅基于词频统计

**目标方案**: Word2Vec + Transformer 动态语境化

1. **Word2Vec 静态词嵌入**
   - 训练中文仙侠语料库
   - 捕捉词义相似度（如"剑宗"≈"门派"）
   - 解决一词多义问题

2. **Transformer 动态语境化**
   - BERT 中文预训练模型
   - Sentence-BERT 句向量
   - 理解上下文语境

3. **混合嵌入策略**
   - TF-IDF（快速粗排）+ BERT（精排）
   - 分层检索，平衡性能与准确度

**预期效果**:
- 语义检索准确率提升 50%+
- 支持同义词、近义词检索
- 理解"苍澜剑宗"和"剑宗"的关系

**技术栈**:
- sentence-transformers 库
- paraphrase-multilingual-MiniLM-L12-v2（多语言支持）
- 本地模型部署 or OpenAI Embedding API
- **同步**: 自动同步 + 向量缓存
