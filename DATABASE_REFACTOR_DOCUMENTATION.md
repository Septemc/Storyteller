# Storyteller 数据库重构文档

## 📋 项目概述

本文档记录了 Storyteller 项目的数据库重构和权限系统配置过程。重构包括用户ID格式标准化、管理员账户创建、权限系统配置和数据隔离实现。

## 🚀 重构目标

1. **用户ID格式重构**：移除"user_"前缀，统一用户ID格式
2. **管理员账户创建**：创建三个具有不同权限级别的管理员账户
3. **权限系统配置**：实现严格的数据访问控制
4. **数据隔离验证**：确保权限配置正确生效

## 🔧 技术实现

### 1. 数据库重构脚本

**脚本文件**：`database_refactor.py`

**主要功能模块**：

#### 🔍 数据库分析 (`check_db.py`)
- 分析当前数据库结构和用户ID格式
- 识别数据关联问题
- 统计各表数据量

#### 🔄 用户ID格式重构
- 移除users表中的"user_"前缀
- 统一用户ID格式为纯UUID字符串
- 保持数据关联完整性

#### 👥 管理员账户创建
- 创建三个管理员账户：
  - `admin001` (ID: `a00000000001`)
  - `admin002` (ID: `a00000000002`) 
  - `admin003` (ID: `a000000000003`)
- 使用安全密码哈希算法

#### 🔐 权限系统配置
- 配置数据访问权限：
  - `admin001`：拥有所有数据的完全访问权限
  - `admin002` 和 `admin003`：无数据访问权限
  - 其他用户：只能访问自己的数据

#### ✅ 验证和测试
- 验证用户ID格式重构结果
- 测试数据隔离效果
- 生成详细重构报告

### 2. 重构后的数据库结构

#### 📊 用户表 (`users`)
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    user_id VARCHAR UNIQUE NOT NULL,  -- 移除"user_"前缀的UUID
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR NOT NULL,
    -- ... 其他字段
);
```

#### 🔗 数据关联表
所有数据表都使用整数ID关联用户，确保数据一致性：
- `worldbook_entries`
- `dungeons` 
- `character_templates`
- `characters`
- `global_settings`
- `story_segments`
- `session_state`
- `presets`
- `llm_configs`
- `regex_profiles`

## 🔒 安全特性

### 1. 密码安全
- 使用bcrypt算法进行密码哈希
- 密码复杂度要求
- 防止彩虹表攻击

### 2. 权限控制
- 基于角色的访问控制 (RBAC)
- 严格的数据隔离
- API级别的权限验证

### 3. 数据保护
- 敏感数据加密存储
- 访问日志记录
- 防止SQL注入攻击

## 📋 使用说明

### 1. 管理员账户登录

**完全权限管理员**：
- 用户名：`admin001`
- 密码：`admin010203`
- 权限：可以访问和操作所有数据

**受限管理员**：
- 用户名：`admin002` 或 `admin003`
- 密码：`admin010203`
- 权限：无法访问任何用户数据

### 2. 数据访问规则

**普通用户**：
- 只能访问自己创建的数据
- 无法查看其他用户的数据
- 数据隔离严格实施

**管理员权限**：
- `admin001`：超级管理员，拥有所有权限
- `admin002/admin003`：受限管理员，无数据访问权限
